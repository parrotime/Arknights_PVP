# 明日方舟干员斗蛐蛐 Demo 设计文档

## 1. 项目目标

本项目是一个本地运行的明日方舟干员自动战斗模拟 Demo。玩法灵感来自“斗蛐蛐”类短视频：两个干员在正方形竞技场内以圆形刚体形式移动、碰撞、反弹，并根据各自的属性、技能和战斗状态自动决出胜负。

第一阶段目标不是还原完整明日方舟战斗系统，而是做出一个可观看、可调参、可扩展的战斗沙盒：

- 干员以圆形代替头像或模型，圆内显示干员名字。
- 圆形在正方形场地内持续运动。
- 干员撞墙会弹性反弹。
- 干员之间碰撞时会弹性反弹，并触发伤害结算。
- 每个干员拥有生命值、攻击力、防御力、速度、技能回复速度等属性。
- UI 显示类似明日方舟风格的蓝色生命条和绿色技能条。
- 技能条按时间自动累积，达到 100% 后自动释放技能。
- 一方生命值归零后战斗结束。

## 2. 技术选型

### 2.1 前端框架

第一版建议不使用 React、Vue 等前端框架，直接采用：

- Vite
- TypeScript
- Canvas
- 普通 DOM UI

原因：

- 游戏主体是实时模拟和逐帧绘制，Canvas 更适合处理竞技场、圆形干员、碰撞反馈和技能特效。
- 第一版 UI 较简单，只需要血条、技能条、按钮、干员选择和战斗日志，普通 DOM 足够。
- 不引入框架可以降低复杂度，让核心精力放在物理、技能和战斗循环上。
- 后续如果要扩展干员图鉴、阵容编辑器、复杂设置页、历史战绩等功能，再引入 React 或 Vue 也不晚。

### 2.2 物理系统

建议使用 Matter.js 负责刚体和碰撞：

- 圆形干员刚体。
- 竞技场墙壁刚体。
- 弹性碰撞。
- 速度、位置、碰撞事件。

游戏自身负责战斗规则：

- 当前生命值。
- 当前技能值。
- 技能释放。
- 伤害计算。
- Buff 和 Debuff。
- 战斗结束判定。

这样可以把“物理运动”和“游戏逻辑”拆开，后续更容易维护。

## 3. 游戏画面设计

### 3.1 主画面布局

建议第一版采用如下结构：

- 页面中央：正方形竞技场。
- 竞技场内：两个圆形干员。
- 页面顶部或竞技场两侧：双方状态面板。
- 页面底部：控制按钮和战斗日志。

状态面板显示：

- 干员名称。
- 生命条，使用蓝色。
- 技能条，使用绿色。
- 当前技能名称。
- 当前生命值和技能值。

控制按钮：

- 开始。
- 暂停。
- 重新开始。
- 战斗速度：x1、x2、x4。
- 可选：切换双方干员。

### 3.2 干员表现

头像资源未准备前，干员用圆形表示：

- 圆形颜色根据职业区分。
- 圆心显示干员名字。
- 圆形边缘可以在受击或释放技能时闪烁。
- 释放技能时在干员或敌人位置显示简单特效。

职业颜色示例：

- 近卫：红色。
- 重装：深灰或钢蓝。
- 术师：紫色。
- 狙击：橙色。
- 医疗：绿色。
- 特种：青色。

## 4. 战斗规则

### 4.1 移动与碰撞

每个干员在战斗开始时拥有初始位置和速度。

基础规则：

- 干员在竞技场内持续移动。
- 撞到墙壁后按弹性碰撞规则反弹。
- 撞到敌方干员后，双方按弹性碰撞规则反弹。
- 干员之间发生碰撞时触发普通攻击伤害。

为了避免两个圆形短时间贴在一起导致连续掉血，需要设置碰撞伤害冷却。

建议规则：

- 每对干员碰撞伤害冷却为 0.5 秒。
- 冷却期间仍然进行物理反弹，但不重复造成碰撞伤害。

### 4.2 普通伤害

普通碰撞伤害建议使用简单公式：

```ts
damage = Math.max(1, attacker.attack - defender.defense)
```

第一版中，双方碰撞时可以互相造成一次普通伤害。

后续可以加入：

- 暴击率。
- 闪避率。
- 职业克制。
- 护甲穿透。
- 法术抗性。
- 撞击速度影响伤害。

### 4.3 技能值

技能值按时间自动回复：

```ts
operator.sp += operator.spRegen * deltaTime
```

当技能值达到最大值时：

- 自动释放技能。
- 技能值清零。
- 触发技能效果和视觉反馈。

第一版统一设定技能最大值为 100，后续可以为每个技能单独配置最大 SP。

### 4.4 战斗结束

当某一方生命值小于等于 0：

- 战斗暂停。
- 显示胜利者。
- 禁止继续触发技能和伤害。
- 用户可以点击重新开始。

如果后续需要，可以加入时间限制和平局判定。

## 5. 干员数据设计

干员数据建议先存储在 TypeScript 文件中，而不是 JSON。

原因：

- TypeScript 可以提供类型检查。
- 技能可以通过 `skillId` 关联到技能逻辑。
- 早期迭代更方便，不需要额外数据加载流程。
- 未来如需编辑器或外部数据源，可以再迁移到 JSON、SQLite 或后端接口。

### 5.1 干员静态定义

```ts
export type OperatorRole =
  | "guard"
  | "defender"
  | "caster"
  | "sniper"
  | "medic"
  | "specialist";

export interface OperatorDefinition {
  id: string;
  name: string;
  role: OperatorRole;
  maxHp: number;
  attack: number;
  defense: number;
  radius: number;
  speed: number;
  spRegen: number;
  skillId: string;
}
```

示例：

```ts
export const operators: OperatorDefinition[] = [
  {
    id: "amiya",
    name: "阿米娅",
    role: "caster",
    maxHp: 1200,
    attack: 95,
    defense: 20,
    radius: 28,
    speed: 160,
    spRegen: 18,
    skillId: "spiritBurst",
  },
  {
    id: "chen",
    name: "陈",
    role: "guard",
    maxHp: 1500,
    attack: 115,
    defense: 35,
    radius: 30,
    speed: 175,
    spRegen: 14,
    skillId: "chiXiao",
  },
];
```

### 5.2 技能定义

技能逻辑建议单独存储，通过 `skillId` 与干员关联。

```ts
export interface SkillDefinition {
  id: string;
  name: string;
  description: string;
  maxSp: number;
  activate: (ctx: SkillContext) => void;
}
```

示例：

```ts
export const skills: Record<string, SkillDefinition> = {
  spiritBurst: {
    id: "spiritBurst",
    name: "精神爆发",
    description: "对敌方造成高额法术伤害",
    maxSp: 100,
    activate: ({ self, enemy }) => {
      enemy.takeDamage(self.attack * 2.4, "arts");
    },
  },

  chiXiao: {
    id: "chiXiao",
    name: "赤霄",
    description: "造成伤害并短暂提升自身速度",
    maxSp: 100,
    activate: ({ self, enemy }) => {
      enemy.takeDamage(self.attack * 1.8, "physical");
      self.addBuff({ type: "speed", value: 1.35, duration: 3 });
    },
  },
};
```

### 5.3 运行时状态

干员静态数据和战斗中的实时状态需要分开。

静态数据：

- 名字。
- 职业。
- 最大生命值。
- 基础攻击。
- 基础防御。
- 基础速度。
- 技能 ID。

运行时状态：

- 当前生命值。
- 当前技能值。
- 当前坐标。
- 当前速度。
- 当前 Buff。
- 当前碰撞冷却。
- 是否存活。

这样可以保证同一个干员定义能够被多场战斗复用。

## 6. 技能系统设计

第一版技能不需要太复杂，但要保证不同干员看起来有差异。

建议先实现以下五类技能：

### 6.1 爆发伤害

立即对敌方造成高额伤害。

适合：

- 术师。
- 近卫。
- 狙击。

### 6.2 回血

恢复自身一定生命值。

适合：

- 医疗。
- 重装。

### 6.3 护盾

在数秒内降低受到的伤害。

适合：

- 重装。
- 辅助型干员。

### 6.4 加速

在数秒内提高移动速度。

适合：

- 近卫。
- 特种。

### 6.5 击退

对敌方造成伤害，并强制改变敌方速度方向。

适合：

- 特种。
- 重装。

## 7. 推荐目录结构

```txt
src/
  main.ts
  style.css
  game/
    Game.ts
    OperatorRuntime.ts
    physics.ts
    skills.ts
    operators.ts
    types.ts
  ui/
    renderer.ts
    dom.ts
```

职责说明：

- `main.ts`：项目入口，创建游戏实例并绑定 UI。
- `Game.ts`：主循环、战斗开始、暂停、重开、胜负判定。
- `OperatorRuntime.ts`：干员运行时状态和行为。
- `physics.ts`：Matter.js 世界、墙壁、圆形刚体、碰撞事件。
- `skills.ts`：技能定义和技能释放逻辑。
- `operators.ts`：干员静态数据。
- `types.ts`：公共类型。
- `renderer.ts`：Canvas 绘制。
- `dom.ts`：血条、技能条、按钮和日志更新。

## 8. MVP 开发范围

第一版建议只做以下内容：

- Vite + TypeScript 项目初始化。
- 一个正方形竞技场。
- 两个圆形干员。
- Matter.js 圆形刚体和墙壁碰撞。
- 干员之间弹性碰撞。
- 碰撞时双方互相造成普通伤害。
- 蓝色生命条。
- 绿色技能条。
- 技能值随时间自动增长。
- 技能满后自动释放。
- 至少两个干员：阿米娅和陈。
- 战斗结束显示胜利者。
- 重新开始按钮。
- 战斗速度切换。

暂不做：

- 联机。
- 干员头像下载。
- 复杂干员养成。
- 完整明日方舟数值还原。
- 后端服务。
- 排行榜。

## 9. 后续扩展方向

### 9.1 干员扩展

加入更多干员，并为每个干员设计独特技能。

可以逐步扩展：

- 星熊：高防御、护盾、反伤。
- 能天使：高攻速、多段伤害。
- 塞雷娅：回血、减伤。
- 银灰：大范围高额爆发。
- 傀影：分身或瞬移。

### 9.2 表现扩展

- 干员头像。
- 技能特效。
- 受击飘字。
- 战斗日志。
- 胜利动画。
- 音效。
- 背景图。

### 9.3 规则扩展

- 多人混战。
- 2v2。
- 随机事件。
- 地形障碍物。
- 职业克制。
- 装备或模组。
- 干员等级和潜能。

### 9.4 工具扩展

- 干员数据编辑器。
- 技能编辑器。
- 战斗录像。
- 自动模拟 N 场并统计胜率。
- 导出战斗结果。

## 10. 第一阶段实现优先级

建议按以下顺序实现：

1. 搭建 Vite + TypeScript 项目。
2. 创建 Canvas 和基础页面布局。
3. 接入 Matter.js，完成圆形和墙壁反弹。
4. 创建干员静态数据。
5. 创建干员运行时状态。
6. 实现碰撞伤害。
7. 实现生命条和技能条。
8. 实现技能自动充能和释放。
9. 实现胜负判定和重新开始。
10. 添加战斗速度控制和简单日志。

第一阶段完成后，项目就已经具备可观看的核心体验，可以开始调数值和增加干员。
